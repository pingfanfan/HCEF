# HCEF-LLM: 大语言模型人本评估框架

[English](README.md) | [中文](README.zh.md)

## 🎯 项目概述

HCEF-LLM（以人为本的大语言模型评估框架）是一个综合性评估框架，旨在从以人为中心的角度评估大语言模型作为"理想智能伙伴"的能力。本项目包含一个交互式的基于网页的排行榜和可视化平台，用于比较模型在多个维度上的性能。

## 🏗️ 框架结构

### 理想智能伙伴 (IIP) 功能角色

1. **IIP-F1: 精确指令与约束理解**
   - 对指令进行细致解读
   - 理解显性和隐性约束
   - 关键方面：
     - 准确解读显性指令
     - 识别隐性约束和假设
     - 明确范围并消除歧义

2. **IIP-F2: 深层目标与意图理解**
   - 洞察根本目的
   - 推断字面请求背后的用户意图
   - 关键方面：
     - 把握最终目标
     - 推断字面请求之外的用户意图
     - 多重或冲突目标的优先级处理

3. **IIP-F3: 高效与适应性任务执行**
   - 有效且高效的任务执行
   - 在动态环境中展现韧性和适应性
   - 关键方面：
     - 有效的任务分解和战略规划
     - 智能的资源分配和工具使用
     - 健壮的异常处理和动态适应

4. **IIP-F4: 质量保证的成果交付与负责任的任务结束**
   - 产出高质量的交付成果
   - 全面且负责任的任务完成
   - 关键方面：
     - 遵循交付标准和格式
     - 输出自我评估和迭代优化

### 核心能力维度 (CD)

核心能力维度（CDs）是实现熟练且人本化表现所需的基本能力，源自智能的基础特征。这些维度被组织为六个主要类别，每个类别都包含特定的子维度：

1. **CD1: 输入处理与理解**
   准确获取信息和深度理解的基础能力
   - **CD1.1: 多模态信息获取**
     - 处理多种输入模态（文本、视觉、听觉）
     - 后续处理的初始入口
     - 全面的输入解释
   - **CD1.2: 上下文与意图理解**
     - 深层上下文关联把握
     - 用户意图推断
     - 语用细节理解
     - 隐含假设识别

2. **CD2: 知识保持与应用**
   信息存储、检索和有效利用的机制
   - **CD2.1: 动态工作记忆**
     - 主动信息维持
     - 上下文跟踪能力
     - 多步推理支持
     - 复杂指令处理
   - **CD2.2: 可访问的长期知识**
     - 庞大知识库访问
     - 事实信息检索
     - 程序性知识应用
     - 常识理解

3. **CD3: 逻辑推理与问题解决**
    用于消除歧义、规划和方案评估的关键能力
    - **CD3.1: 分析与推理能力**
      - 用于模式识别的归纳推理
      - 用于规则应用的演绎推理
      - 基于证据的推理生成
      - 逻辑结果推导
    - **CD3.2: 结构化问题分解**
      - 将复杂问题分解为子组件
      - 组件关系分析
      - 系统解决方案制定
      - 战略执行规划

4. **CD4: 想象与创造性认知**
    用于产生新颖想法和抽象思维的非标准思维
    - **CD4.1: 预测与生成性前瞻**
      - 未来状态预测
      - 结果预测建模
      - 场景生成
      - 基于模式的补全
    - **CD4.2: 新颖解决方案构思与抽象表征**
      - 发散思维应用
      - 类比联系建立
      - 抽象概念表征
      - 非常规解决方案生成

5. **CD5: 以人为中心与伦理对齐**
    理解并遵循人类价值观和社会规范
    - **CD5.1: 同理心与社会意识**
      - 情感线索识别
      - 社会动态解读
      - 人际细微差别理解
      - 社会智能交互
    - **CD5.2: 价值理解与伦理考量**
      - 人类价值观理解
      - 伦理原则应用
      - 负责任AI框架对齐
      - 伦理情境识别

6. **CD6: 输出生成与交付**
    生成清晰、恰当且有效的输出
    - **CD6.1: 清晰连贯的沟通**
      - 语法正确性
      - 逻辑结构形成
      - 信息清晰度优化
      - 信息传递有效性
    - **CD6.2: 适应性与目标导向表达**
      - 情境适配的风格调整
      - 针对受众的语气调整
      - 工具使用有效性
      - 行动执行精确性

### 熟练度描述符 (PD)

#### PD1: 新兴
- **特征**：基本功能，存在显著限制
- **表现**：
  - 只能处理简单、明确的任务
  - 需要详细的指导和结构化输入
  - 输出质量不稳定
  - 有限的错误处理能力

#### PD2: 发展中
- **特征**：能力提升，存在一些约束
- **表现**：
  - 可以处理中等复杂度的任务
  - 需要适度的指导
  - 输出质量逐步提高
  - 基本的错误处理能力

#### PD3: 熟练
- **特征**：稳定性能，满足大多数要求
- **表现**：
  - 能够处理复杂任务
  - 较强的自主性
  - 输出质量稳定
  - 良好的错误处理能力

#### PD4: 专家
- **特征**：卓越性能，超越预期
- **表现**：
  - 能够处理高度复杂的任务
  - 极强的自主性和创造力
  - 输出质量优异
  - 出色的错误处理和恢复能力

## 📊 评估结果

排行榜包含了多个最先进的语言模型的评估结果：

- OpenAI o4 mini
- OpenAI GPT 4.1
- Claude 4 Sonnet
- Gemini 2.5 Pro
- Qwen3 235B

每个模型都在所有六个能力维度上进行评估，并被分配一个整体熟练度等级。
